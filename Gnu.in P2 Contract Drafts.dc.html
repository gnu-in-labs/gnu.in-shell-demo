A contracts/surfaces.toml — native migration state + topbar→taskbar rename
Today the contract records only owner_qml for ~18 surfaces while engine/gnuin-* holds a native Rust host for nearly all of them; only shellSettings has owner_rs. With QML retired-as-tech, a reviewer can't tell which surfaces have a native replacement ready. Add three keys per surface — and fix the naming: the surface is [surfaces.taskbar] but its namespace, selector, and host all say topbar.
rename · topbar → taskbar everywhere. Current real identifiers gnuin-topbar (namespace), shell.topbar.host (selector), TopbarHost.qml (host) are inconsistent with the canonical taskbar surface name. This is a real cross-repo change: grep topbar across gnu.in-os + gnu.in-shell, update the schema enum for the selector, and rename the QML host file. Do it as one commit so the contract and call sites stay in lockstep.

C Decision memo — chrome icon family
The design system (TASTE.md / README / SKILL) mandates Material Symbols Rounded for chrome, "confirmed in the real Appearance.qml" — i.e. the QML shell. But the forward native bar engine/gnuin-bar/src/text.rs loads a Nerd-Font (Material Design Icons patch) face and ships nf-md-* codepoints (WIFI/BLUETOOTH/VOLUME/BATTERY/CLOCK). The two don't match, and the stated authority points at retiring tech.
OPTION 1
Standardize on Nerd-Font MDI
Native bar already ships it; one static font file, rasters cleanly in fontdue. Update the design docs. Least code churn.
OPTION 2
Standardize on Material Symbols
Matches the stated brand, but the variable FILL/wght axes are awkward to raster in fontdue; the native bar must switch faces. More work, retiring-era choice.
OPTION 3 ✓ rec
Split by layer
Identity = cube family (unchanged). Native chrome = Nerd-Font MDI. Keep Material Symbols only where a QML fallback still renders. Retire the "Material Symbols = chrome" guidance as QML-era.
recommendation — Option 3. The native renderer already ships Nerd Font; Material Symbols Rounded was the Quickshell shell's family and retires with it. Needs sign-off, then update TASTE.md §4 + README §Iconography + SKILL.md to name the chrome icon family for native hosts.
