CRITICAL
SKILL.md hard-rules flat bézier motion as universal — TASTE.md requires M3-Expressive springs for chrome
SKILL.md hard-rules give no distinction between chrome and identity layer motion. TASTE.md §2 establishes that chrome surfaces must use M3-Expressive springs with intentional overshoot (--ease-expressive), while the old calm bézier belongs only on the identity/terminal layer. A move with no overshoot "reads as a different, lesser OS."
Motion default ease cubic-bezier(0.2, 0.7, 0.2, 1); durations 120 / 200 / 360ms; listening pulse 1400ms.
(no chrome / identity distinction given)
The shell runs M3-Expressive springs, not flat easings. Spatial moves use --ease-expressive / --ease-expressive-fast — the overshoot is intentional and is half the personality.
Reserve the calm --ease-standard for the identity/terminal layer.
Add two-layer motion table to SKILL.md hard rules: chrome → --ease-expressive (overshoot); identity/terminal → cubic-bezier(0.2,0.7,0.2,1) calm bézier

03 — Live surface bugs (from AUDIT_SURFACES_2026-06-11)
CRITICAL · P1
Taskbar clock renders as "16:2" — Lift.qml / Origami.qml missing implicitWidth propagation
Confirmed in the headless surface audit. The clock chip is too narrow so the text overflows. Root cause: Lift.qml and Origami.qml are bare Item wrappers with no implicitWidth/Height, so the implicit-size chain collapses to 0 and WidgetBase becomes only widgetPadX wide. This is a structural bug — it affects every WidgetBase wrapped in either component, not just the clock.
Horloge clippée au bord droit ("16:2"). Cause racine: WidgetBase.implicitWidth = contentLoader.implicitWidth + padX, or ClockWidget.content = Lift{Origami{Text}} et A.03 Lift / A.07 Origami sont des Item sans implicitWidth → la chaîne implicite s'effondre à 0.
Patch: propager implicitWidth/Height du contenu dans Lift.qml / Origami.qml (fichiers RTTC — réconcilier le contrat).
In Lift.qml and Origami.qml: add `implicitWidth: contentItem.implicitWidth` and `implicitHeight: contentItem.implicitHeight` (or equivalent child binding); reconcile RTTC spec for both
