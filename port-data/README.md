# gnu.in-OS · port-data

Golden data for the **gnuin-compose-core** Rust port, generated by **Central**
(the live simulator). The simulator and this corpus share one model, so a fixture
here is exactly what Central renders on screen.

```
port-data/
├─ scene_types.rs     · Rust type definitions (NodeKind, Band, NodePayload, Scene, Envelope…)
├─ tokens.json        · resolved token bridge (palette · radii · spacing · springs · durations · layer bands)
├─ MANIFEST.json      · M6 surface-proof rail (status + proof per surface)
└─ scenes/
   ├─ scene.idle.json            · chrome only (5 nodes)
   ├─ scene.launcher.json        · launcher open
   ├─ scene.sidebar.json         · sidebar open
   ├─ scene.osd.json             · OSD volume
   ├─ scene.modal.json           · modal + scrim
   ├─ scene.notification.json    · notification stack
   ├─ scene.context-menu.json    · context-menu at cursor
   ├─ scene.full.json            · every surface composed
   └─ scene.launcher.motion.json · launcher mid-open, carries a MotionTick damage record
```

## Scene fixture shape

```json
{
  "id": "scene.launcher",
  "nodes": [
    { "id": 0, "kind": "Background", "interactive": false, "layer": 0,
      "bounds": [0, 0, 1280, 800], "payload": { "Fill": "0x141313FF" } }
  ],
  "damage": []
}
```

`bounds` is `[x, y, w, h]` in logical px (1280×800 reference). `layer` is the
canonical `NodeKind::band()` value. `kind` strings map 1:1 onto the `NodeKind`
enum in `scene_types.rs`.

## Consuming it from the port

1. `serde_json` the `scenes/*.json` into `Scene` (the field layout matches
   `scene_types.rs`).
2. Resolve a `TokenBundle` per node from `tokens.json` (radius from `radii_px`,
   fill from `palette`, motion from `surface_spring`).
3. Render with the `tiny-skia` `SoftwareRenderer`, then assert the committed
   frame + `hyprctl` layer-record against `MANIFEST.json` — the same gate
   `tools/verify-surface-proof.py` runs.

The non-interactive kinds (`Osd`, `Notification`, `ScreenCorner`, `Background`,
`Wallpaper`) must be absent from the hit / input_region: a click at any point
routes to the topmost **interactive** node only.
